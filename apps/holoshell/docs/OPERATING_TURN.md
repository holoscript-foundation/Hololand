# HoloShell Operating Turn

The operating turn is the repeatable local loop for Brittney-in-HoloShell:

```powershell
pnpm run holoshell:operating-turn
```

It first reconciles stale run registry records, then refreshes process health,
hardware reality, legacy windows, build custody, run custody, legacy app
absorption, readiness evidence, the operator brief, Brittney's custody action,
and the visual witness.

The turn is read-only by default. It records evidence and blocks process kills,
legacy app mutation, file deletion, and build termination behind preflight and
break-glass policies.

Registry reconciliation is non-destructive. It clears phantom active runs from
HoloShell's own registry only when the expected end time has passed and no
visible PID exists.

Process health runs before hardware reality so lane ownership evidence from
local process ancestors and legacy app windows is fresh before run custody and
the operator brief count owner-unknown runs.
