## Part 3: Platform Economics

### Why Platform Beats Product

**Gross Margin Comparison**:

| Model | COGS | Gross Margin | Scale Curve |
|-------|------|--------------|-------------|
| Consumer SaaS | 20-30% | 70-80% | Linear |
| **Platform** | **1-5%** | **95-99%** | **Exponential** |

**Platform COGS Breakdown** (at 10K white-label customers):
- Infrastructure: $15K/mo ($1.50/customer)
- Support: $10K/mo ($1.00/customer)
- **Total**: $25K/mo ($2.50/customer)

**Revenue per White-Label Customer**: $500-$5,000/mo
**Platform Margin**: 95-99% (vs 70-80% for consumer product)

### Network Effects

```
More Creators → More Templates → More Users → More Revenue
    ↑                                              ↓
    └──────────── Marketplace Flywheel ────────────┘
```

**Compounding Value**:
1. **Creator uploads template** → Earns 80% revenue share
2. **User buys template** → Platform earns 20% fee
3. **More users** → More template demand → More creators join
4. **White-label customers** → Use marketplace templates → Creators earn more
5. **Self-sustaining growth** without linear hiring

## Part 8: Technical Architecture

### Multi-Tenant Design

**Database**: PostgreSQL with row-level security (RLS)

```sql
-- Tenant isolation at row level
CREATE POLICY tenant_isolation ON user_worlds
  USING (tenant_id = current_setting('app.current_tenant')::uuid);

-- Each white-label customer = unique tenant_id
-- Guarantees data isolation, GDPR compliance
```

**Infrastructure**: Kubernetes with namespace per tenant

```yaml
# Each tenant gets isolated namespace
apiVersion: v1
kind: Namespace
metadata:
  name: tenant-acme-corp
  labels:
    tenant: acme-corp
    tier: enterprise
```

**Scaling**:
- Shared control plane (API, auth, billing)
- Isolated data plane (worlds, assets, users)
- Auto-scaling: 1-1000 concurrent users per tenant

### API v1.0 Endpoints

```
POST /api/v1/worlds          # Create new world from template
GET  /api/v1/worlds/:id      # Get world data
PUT  /api/v1/worlds/:id      # Update world
POST /api/v1/worlds/:id/fork # Remix template

GET  /api/v1/templates       # Browse marketplace
POST /api/v1/templates       # Submit template (creators)

POST /api/v1/tenants         # Create white-label customer
GET  /api/v1/analytics       # Usage metrics (enterprise)
```

**Rate Limits**:
- Free tier: 100 req/hour
- Creator tier: 1,000 req/hour
- Pro tier: 10,000 req/hour
- Enterprise: Custom (no limit)

### Layer 3: x402 AI Agent Payment Protocol

**Protocol**: HTTP 402 (Payment Required) via Coinbase Agentic Wallets
**Launch**: February 11, 2026 (Coinbase)
**Scale**: 50M+ transactions projected by Coinbase

#### Value Proposition

AI agents (OpenAI, Anthropic, etc.) can pay per API call to programmatically generate 3D worlds, unlocking the "machine customer" market (projected 10-30% of SaaS revenue by 2027-2028).

**How It Works**:
```
AI Agent sends request:
  POST /api/v1/generate-world
  { prompt: "medieval castle with dragon", format: "holoscript" }

Hololand API responds:
  HTTP 402 Payment Required
  { amount: "$0.50", payment_address: "0x...", session_id: "abc123" }

AI Agent wallet (Coinbase) sends gasless payment:
  → Coinbase subsidizes gas fees
  → Trusted Execution Environment (TEE) prevents prompt injection
  → Payment confirmed in <2 seconds

Hololand API processes payment:
  → Generates HoloScript world via GPT-4 + compiler
  → Returns world file + metadata

AI Agent receives:
  200 OK
  { world_url: "https://hololand.io/worlds/abc123.holo" }
```

#### Technical Implementation

**x402 Endpoint** (`POST /api/v1/generate-world`):
```typescript
import { CoinbaseWallet } from '@coinbase/wallet-sdk';

export async function POST(req: Request) {
  const { prompt, format = 'holoscript' } = await req.json();

  // Step 1: Check if payment header exists
  const paymentHeader = req.headers.get('x-payment-token');

  if (!paymentHeader) {
    // Return 402 Payment Required
    return new Response(JSON.stringify({
      error: 'Payment required',
      amount_usd: 0.50,
      payment_address: process.env.HOLOLAND_PAYMENT_WALLET,
      session_id: generateSessionId(),
      supported_protocols: ['x402', 'coinbase-agentic']
    }), {
      status: 402,
      headers: { 'Content-Type': 'application/json' }
    });
  }

  // Step 2: Verify payment via Coinbase API
  const paymentValid = await verifyCoinbasePayment(paymentHeader);

  if (!paymentValid) {
    return new Response(JSON.stringify({
      error: 'Invalid payment'
    }), { status: 403 });
  }

  // Step 3: Generate world using HoloScript compiler
  const worldScript = await generateWorldFromPrompt(prompt);
  const compiled = await compileHoloScript(worldScript, format);

  // Step 4: Store world in database
  const world = await db.worlds.create({
    source: 'ai_agent',
    prompt: prompt,
    script: worldScript,
    compiled: compiled,
    payment_tx: paymentHeader
  });

  // Step 5: Return world URL
  return new Response(JSON.stringify({
    world_id: world.id,
    world_url: `https://hololand.io/worlds/${world.id}`,
    format: format,
    script: worldScript,
    compiled: compiled
  }), {
    status: 200,
    headers: { 'Content-Type': 'application/json' }
  });
}
```
