## Appendix A: Technical Deep Dive

### Multi-Tenant Architecture

**Database**: PostgreSQL with row-level security

```sql
-- Tenant isolation
CREATE POLICY tenant_isolation ON user_worlds
  USING (tenant_id = current_setting('app.current_tenant')::uuid);
```

**Infrastructure**: Kubernetes with namespace per tenant

```yaml
apiVersion: v1
kind: Namespace
metadata:
  name: tenant-stanford-edu
  labels:
    tenant: stanford
    tier: pro
```

**Scaling**: 1-1000 concurrent users per tenant, auto-scaling

### API v1.0 Design

```typescript
// Core endpoints
POST   /api/v1/worlds          // Create from template
GET    /api/v1/worlds/:id      // Get world data
PUT    /api/v1/worlds/:id      // Update world
POST   /api/v1/worlds/:id/fork // Remix template

// Marketplace
GET    /api/v1/templates       // Browse marketplace
POST   /api/v1/templates       // Submit (creators)

// White-label
POST   /api/v1/tenants         // Create customer
GET    /api/v1/analytics       // Usage metrics
```

**Rate Limits**:
- Free: 100 req/hour
- Creator: 1,000 req/hour
- Pro: 10,000 req/hour
- Enterprise: Unlimited

**SLA**: 99.9% uptime (enterprise tier)
