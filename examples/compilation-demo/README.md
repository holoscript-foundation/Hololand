# 🚀 HoloScript Interactive Compilation Demo

**Live demonstration of HoloScript's "Write Once, Deploy Everywhere" cross-platform compilation.**

This interactive web tool showcases how a single HoloScript file compiles to multiple game engines and platforms.

---

## ✨ Features

- **Monaco Editor** - Full-featured code editor with syntax highlighting
- **Live Compilation** - See generated code instantly
- **5 Platform Targets**:
  - Unity (C#)
  - Unreal Engine (C++)
  - Godot (GDScript)
  - Babylon.js (TypeScript)
  - WebGPU (WGSL shaders)
- **Code Download** - Download generated code for each platform
- **Side-by-Side View** - Compare HoloScript source with compiled output
- **Real-time Stats** - See code expansion metrics

---

## 🚀 Quick Start

### Install Dependencies

```bash
pnpm install
```

### Run Development Server

```bash
pnpm dev
```

The demo will open at `http://localhost:3001`

### Build for Production

```bash
pnpm build
pnpm preview
```

---

## 💡 How to Use

1. **Edit HoloScript** - Modify the code in the left panel
2. **Select Target** - Choose Unity, Unreal, Godot, Babylon, or WebGPU
3. **Compile** - Click the "⚡ Compile" button
4. **View Output** - See the generated platform-specific code
5. **Download** - Click "⬇️ Download" to save the generated code

---

## 📝 Example HoloScript

```holoscript
@zone "Demo Scene" category:"demo" maxPlayers:20

composition "MyScene" {
  // Spinning cube with glow
  object "MagicCube" {
    @spatial @rotate @emissive
    geometry: "cube"
    position: [0, 2, 0]
    material: {
      color: "#ff00ff"
      emissive: "#ff00ff"
      emissiveIntensity: 0.5
    }
  }

  // Floating sphere
  object "FloatingSphere" {
    @float @pulse
    geometry: "sphere"
    position: [3, 2, 0]
  }

  // Portal to another zone
  portal "Exit" {
    @interactive
    position: [0, 1, -5]
    destination: "main_plaza"
  }

  // Lighting
  light "Sun" {
    type: "directional"
    intensity: 1.2
  }
}
```

## 🎯 What Gets Compiled

The demo generates **production-ready code** for each platform:

### Unity (C#)
- MonoBehaviour components
- GameObject creation
- Material setup with PBR
- Trait components
- Lighting configuration

### Unreal (C++)
- AActor classes
- Procedural mesh generation
- MaterialInstanceDynamic creation
- Component architecture
- Units converted to centimeters

### Godot (GDScript)
- Node3D scene structure
- StandardMaterial3D setup
- Metadata-based traits
- Light nodes

### Babylon.js (TypeScript)
- Scene class structure
- MeshBuilder API
- StandardMaterial with PBR
- Runtime trait application

### WebGPU (WGSL)
- Vertex shaders
- Fragment shaders with PBR
- Uniform buffer structures
- Cook-Torrance BRDF lighting

## 📊 Metrics

**Code Expansion**:
- HoloScript: ~50 lines (declarative)
- Unity: ~150-200 lines (imperative C#)
- Unreal: ~200-250 lines (imperative C++)
- Godot: ~120-150 lines (imperative GDScript)
- Babylon: ~150-180 lines (TypeScript)
- WebGPU: ~200+ lines (WGSL shaders)

**Average expansion ratio**: **3-4x**
**Maintenance reduction**: **66-80%** (5 codebases → 1)
