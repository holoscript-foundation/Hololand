## How This Works

This file is the **single source of truth** for strategic decisions at **extreme code velocity**.

**Purpose**:
- Avoid repeated questions (Decision Bottleneck mitigation)
- Track reversibility and validation criteria
- Enable event-driven context sync
- Document AI vs human stakeholders
- Capture decision rationale for future reference

**Update frequency**: After each decision sprint + ad-hoc for critical decisions

**Decision Laws** (from uAA2++ research):
1. **Reversibility**: Prefer reversible actions
2. **Exploration-Exploitation**: Balance discovery vs delivery
3. **Ownership**: Own decisions explicitly
4. **Workaround Prohibition**: Fix root causes
5. **Fatigue**: Recognize when tired, delegate
6. **Gate**: Break complex tasks into phases
7. **Information-Action**: Gather info before acting

- Fix is <100 lines of code

- No architecture changes

- Performance implications unclear
