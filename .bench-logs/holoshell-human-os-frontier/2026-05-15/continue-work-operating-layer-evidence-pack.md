## Human Job

A non-technical user opens HoloShell and asks: "What is this computer doing, can I safely continue HoloLand work, what agents/apps/files are involved, and what needs approval before anything changes?"

The user should not need Terminal, Git, Task Manager, Chrome internals, `.env`, or repo knowledge. HoloShell should show the work as an operating room: health, source, agents, open apps, pending approvals, receipts, warnings, and replay paths.

## `.hsplus` Concept

State machine:

`boot_shell -> collect_receipts -> classify_machine_state -> validate_sources -> stage_safe_workflow -> require_approval -> witness_projection -> file_gaps -> ready|warn|blocked`

Policy:

- Read-only capture of apps/processes/files is allowed with local receipts.
- Staging a workflow is guarded and nonce-bound.
- Executing staged workflow steps needs fresh user approval and execute-enabled daemon.
- Typing commands, opening apps, media playback, killing PIDs, deleting files, installs, and publishing are guarded or break-glass depending on target.
- Source validation failure blocks "source native" claims even if the HTML projection renders.

Agent roles:

- Codex hardware: local probes, validation, screenshot, HoloMesh task filing.
- Brittney/HoloShell: plain-language plan, risk narration, approval staging.
- HoloScript substrate: schema, parser, receipt, policy, CLI/MCP gaps.
- HoloLand product: room objects, visual state language, first-screen clarity.
- Gemini/browser: visual witness when browser plugin path is healthy.
- HoloMesh: task dedupe, ownership, status broadcasting.

## HoloScript Upstream Recommendations

- Promote a generic `ShellObject`, `PermissionEnvelope`, `WorkflowApprovalReceipt`, and `OperatingLayerReceipt` schema/validator upstream.
- Add a validator that rejects ambiguous approval state labels: staged is not approved, approved is not executed, executed is not verified.
- Expose a local-first graph/codebase receipt when hosted MCP cannot inspect Windows roots.
- Decide whether `spatial_group "Name" at [x,y,z]` is intended syntax. If yes, fix the HoloScript parser; if no, update HoloShell source to current valid syntax and add a contract test.
