## GOTCHAS

### G.SPATIAL.01 - React Context Performance Trap
**Severity**: MEDIUM  
**Symptom**: Re-renders cascade through entire spatial game component tree on every state change, causing framerate drops below 30fps

**Cause**: Single monolithic Context holding all game state (arena data, player positions, AI state, combat log). Any state update triggers all Context consumers to re-render. In spatial games, player position updates 60+ times/second.

**Fix**: Split Context by update frequency:
- `ArenaContext`: Static arena layout (rarely changes)
- `PlayerContext`: Player position/state (updates 60fps)
- `CombatContext`: Combat events (sporadic)

Use `React.memo()` on components consuming slow-changing contexts. Result: 60fps maintained, only position-dependent components re-render on player movement.

### G.SPATIAL.02 - AI Context Window Memory Leak
**Severity**: HIGH  
**Symptom**: Browser tab crashes after 20+ AI content generations, memory usage grows 50MB+ per generation

**Cause**: Unlimited context array accumulation in BrittneyGameIntegration service. Each generation appends to `this.context` array without size limits. After 20 generations, context contains 100KB+ of text, sent with every new request.

**Fix**: Implement sliding context window:
```typescript
const MAX_CONTEXT = 10
if (this.context.length > MAX_CONTEXT) {
  this.context = this.context.slice(-MAX_CONTEXT)
}
```
Keep only last 10 generations in context. Prevents unbounded memory growth while maintaining narrative coherence.

### G.SPATIAL.03 - HoloScript Compile-Time Type Mismatch
**Severity**: LOW  
**Symptom**: HoloScript compiles successfully but runtime errors occur when React components consume compiled output

**Cause**: HoloScript spatial types (e.g., `Zone`, `Spawn`) compile to generic objects. TypeScript React components expect specific interfaces. No bridge validation between HoloScript output and React input types.

**Fix**: Generate TypeScript type definitions during HoloScript compilation:
```typescript
// Generated from BattleArena.hsplus
export interface CompiledBattleArena {
  zones: {
    SafeZone: { x: number, y: number, width: number, height: number }
    CombatZone: { x: number, y: number, width: number, height: number }
  }
  spawns: { PlayerStart: { x: number, y: number, radius: number } }
}
```
Enables TypeScript validation of React components consuming HoloScript output.

### G.SPATIAL.04 - Batch Generation Race Conditions
**Severity**: MEDIUM  
**Symptom**: When generating full quest with multiple NPCs simultaneously, NPC dialogue references NPCs that don't exist yet

**Cause**: Parallel AI generation calls don't share context. Quest generation and 3 NPC generations fire simultaneously. NPC #3 doesn't know about NPC #1 and #2 because their generations haven't completed.

**Fix**: Sequential batch generation with context chaining:
```typescript
async generateFullQuest(params) {
  const quest = await this.generateQuest(params)
  this.context.push(quest.description)
  
  const npcs = []
  for (const npcRole of quest.requiredNPCs) {
    const npc = await this.generateNPC({ role: npcRole })
    this.context.push(npc.dialogue)
    npcs.push(npc)
  }
  
  return { quest, npcs }
}
```
Each generation completes and updates context before next begins.
